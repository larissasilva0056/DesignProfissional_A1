# README -- Materiais de Aula de Banco de Dados

**Disciplina: Banco de Dados** **Professora: Leticia Toled**

Este repositório reúne os materiais de estudo utilizados na disciplina
de Banco de Dados. Aqui registro, como aluna, minha compreensão dos
conteúdos, o propósito de cada material e como eles se conectam no
aprendizado de modelagem e construção de bancos de dados.

## 1. Conteúdos Incluídos

### Aula 2 -- Modelagem de Dados (Modelo Lógico x Modelo Físico)

Nesta aula entendi a diferença entre modelo conceitual, lógico e físico.
Compreendi que:

-   O modelo lógico é a representação estruturada das entidades,
    atributos e relações, sem depender do SGBD.
-   O modelo físico descreve como essas estruturas serão realmente
    implementadas no SGBD escolhido.
-   Também estudei os tipos de bancos NoSQL e quando eles são mais
    adequados em relação aos bancos relacionais.

### Aula 3 -- Modelagem de Dados e Modelo Entidade-Relacionamento

Esta aula introduziu o Modelo Entidade-Relacionamento (MER). Aprendi:

-   O conceito de entidade, seus tipos.
-   Tipos de atributos: simples, compostos, derivados, multivalorados.
-   O conceito de chave e domínios.
-   Tipos de relacionamentos, cardinalidades e restrições.
-   Entidades fracas e relacionamentos de identificação.
-   Como construir um diagrama ER coerente.

### Aula 4 -- Generalização e Especialização

Aprendi como funcionam conceitos avançados do MER. Compreendi:

-   Generalização e especialização.
-   Total, parcial, exclusiva e compartilhada.
-   Entidade associativa.

### Transformação do Modelo ER para Modelo Relacional

Aprendi:

-   Transformar entidades em tabelas.
-   Atributos em colunas.
-   Implementar relacionamentos.
-   Converter generalização/especialização para tabelas.

### Normalização (1FN, 2FN, 3FN)

Aprendi:

-   Dependência funcional.
-   Dependência total, parcial e transitiva.
-   Normalização para eliminar anomalias.
-   Garantia de integridade dos dados.

## 2. Conexão entre os Arquivos

Cada aula representa uma etapa do desenvolvimento de um banco de dados,
desde o entendimento do minimundo até a implementação física e
otimização por normalização.

## 3. Objetivo

Reunir todo o material da disciplina em formato organizado para estudo,
revisões e uso profissional futuro.
